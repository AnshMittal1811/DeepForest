Research Library for segmenting individual trees within LiDAR point clouds

Getting started manual

 

Introduction

The manual describes using a research prototype (developed in Python) as a library (download here) for performing individual tree segmentation within small-footprint airborne LiDAR point cloud of a forest.  A DSM-based method [1] and a multi-story segmentation method [2] can be performed using the library.  The fpclib folder includes the Python source files:

·       util.py including functions for I/O operations such as reading from and writing to disk, and basic visualization.

·       geometry.py including functions for geometrical operations and tests.

·       forest.py including functions for more high-level forest point cloud operations.

·       crownSegmenter.py including the functions dealing with segmentation of individual trees.

·       constants.py including a set of constant variables representing the configurations of the segmentation method, point cloud parameters, etc.

Although very little inline documentation is available, using the library does not require understanding the code at all.  Basic familiarity with Python programming language is needed.

Prerequisits

In addition to standard Python 2.7, numpy, matplotlib, and shapely packages are required.

Initial setup

Basic configuration of the segmentation method and the point cloud parameters can be set in the constants.py.  The most important parameter is NOMINAL_POST_SPACING, which should be set in advance (all of the length metrics are in feet), and the rest of the parameters are set automatically.  Also MINIMUM_DETECTABLE_CROWN_WIDTH (already set to 3.4 ft) is an important parameter, which is better to be set larger.  So, the recommendation is to use the maximum of 4×NOMINAL_POST_SPACING and the smallest crown width that is desired to be detected.  MINIMUM_TREE_HEIGHT was also set to 15.0 ft, which can be modified if desired.

Input file

The input point cloud needs to be in the CSV format, and the first six columns should be x, y, z, class, intensity, and return, respectively (see the samples in the plots folder).  The LiDAR points should already be processed and classified to ground (class equals to  2 or 8) or vegetation  (class equals to 3, 4, or 5).  At least one ground point is required to be present for determining the normalized heights (height above ground) of other points.

Segmenting individual tree crowns

As mentioned, two methods of segmentation can be performed:

1.     DSM-based that only uses the surface data.  This is a pretty robust method and identifies most of the dominant, co-dominant, and intermediate trees with a good precision.  It also identifies some of the overtopped trees.  Identified intermediate and overtopped crowns are usually partial crowns.

2.     Multi-story that layers the point cloud and segments each layer separately using the DSM-based method.  The layering mechanism can be regional or local (please see [2]). Identified tree crowns can then be merged across layers, even though it does not improve segmentation significantlly.  This method remarkably boosst tree identification (specially over-topped ones), yet it increases number of false positives.

Example

1.     run.py performs DSM-based segmentation.  There is inline comments inside showing the usage of the library and how to modify it to perform multi-story segmentation.  Please take a look at them before proceeding.

In the command window, navigate to the main folder of the library (where run.py exists) and type this command:

python run.py plots\122.csv output\122.txt

The Python interpreter (presumably already installed) executes the run.py script.  plots\122.csv shows the path to the input point cloud file, and output\122.txt shows the path to write the output file.  The script will load and segment the point cloud, visualizes the segmented crowns from an aerial view, and writes the result in the provided output file.
